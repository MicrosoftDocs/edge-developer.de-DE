# <a name="javascript-representation-of-windows-runtime-types"></a>JavaScript-Darstellung von Windows-Runtime-Typen  

[!INCLUDE [deprecation-note](../includes/legacy-edge-note.md)]  

In der folgenden Tabelle wird beschrieben, wie JavaScript Windows-Runtime-Typen darstellt.  

> [!IMPORTANT]
> Windows-Runtime-Features sind für Apps, die in Internet Explorer ausgeführt werden, nicht verfügbar.  

## <a name="windows-runtime-types-in-javascript"></a>Windows-Runtime-Typen in JavaScript  

| Windows-Runtime-Typ | JavaScript-Darstellung | Beschreibung |  
|:--- |:--- |:--- |  
| `UInt8` | `Number` | Eine Windows-Runtime `Uint8` ist eine nicht signierte 8-Bit-Ganzzahl, die als JavaScript-Nummer dargestellt wird.  Ein JavaScript-Wert wird zunächst in eine JavaScript-Nummer konvertiert, und dann wird `ToUint32` der Prozess befolgt.  Wenn sich der Wert der JavaScript-Nummer außerhalb des Bereichs eines Uint8 befindet, wird modulo 2^8 angewendet.  |  
| `Int32` | `Number` | Eine Windows-Runtime `Int32` ist eine signierte 32-Bit-Ganzzahl, die als JavaScript-Zahl dargestellt wird.  Ein JavaScript-Wert wird zunächst in eine JavaScript-Nummer konvertiert, und dann wird `ToInt32` der Prozess befolgt.  Wenn sich der Wert der JavaScript-Nummer außerhalb des Bereichs eines befindet, wird `Int32` modulo 2^16 angewendet.  |  
| `Int64` | `Number` | Eine #A0 ist eine signierte ganzzahlige 64-Bit-Zahl, die als Standardnummer dargestellt wird, wenn sie in den Bereich `Int64` \[-2^53, 2^53\] fällt.  Wenn er außerhalb dieses Bereichs liegt, wird er als Zahlenwert dargestellt, der über einen benutzerdefinierten Sicherungsspeicher verfügt, der die vollständigen 64 Bit ganzzahliger Daten verwaltet.  Alle mathematischen Vorgänge für diese benutzerdefinierten Int64-Nummernwerte führen dazu, dass der Wert in eine Standardnummer im Bereich \[-2^53, 2^53\] konvertiert wird.  Wenn sich der Wert außerhalb dieses Bereichs befindet, wird ein Typfehler ausgelöst.  Die Ausnahmen für diesen Konvertierungsprozess sind die Vorgänge , , und, die die Argumente basierend auf den vollständigen 64 Bit vergleichen, wenn zwei dargestellte `==` `===` `SameValue` `RelationalComparison` 64-Bit-Werte übergeben werden.  Wenn es sich bei beiden Argumenten um eine Standardmäßige JavaScript-Nummer handelt, werden diese Vorgänge auch vor dem Vergleich in eine JavaScript-Zahl konvertiert.  Ein #A0 wird direkt zugewiesen, wenn es sich um einen Int64-Wert selbst handelt. Andernfalls wird das Ergebnis der Konvertierung `ToInteger` auf den Wert an die Windows-Runtime übergeben.  Wenn bei der Typkoercion ein Fehler auftritt, wird eine Ausnahme zurückgegeben.  |  
| `Uint64` | `Number` | Eine #A0 ist eine nicht signierte ganzzahlige 64-Bit-Zahl, die als Standardnummer dargestellt wird, wenn sie in den Bereich `Uint64` \[0, 2^53\] fällt.  Wenn sie außerhalb dieses Bereichs liegt, wird sie als Zahl dargestellt, die über einen benutzerdefinierten Sicherungsspeicher verfügt, der die vollständigen 64 Bit nicht signierter ganzzahliger Daten verwaltet.  Alle mathematischen Vorgänge für diese benutzerdefinierten Uint64-Werte führen dazu, dass der Wert in eine Standardnummer im Bereich \[0, 2^53\] konvertiert wird.  Wenn sich der Wert außerhalb dieses Bereichs befindet, wird ein Typfehler ausgelöst.  Die Ausnahmen bei diesem Konvertierungsprozess sind die Vorgänge , , und, die die Argumente basierend auf den vollständigen 64 Bit vergleichen, wenn zwei `==` `===` `SameValue` `RelationalComparison` 64-Bit-Werte übergeben werden.  Wenn es sich bei beiden Argumenten um eine Standardmäßige JavaScript-Nummer handelt, werden diese Vorgänge auch vor dem Vergleich in eine JavaScript-Zahl konvertiert.  Ein #A0 wird direkt zugewiesen, wenn es sich um einen Uint64-Wert selbst handelt. Andernfalls wird das Ergebnis der Anwendung der Konvertierung auf den Wert, gefolgt von `ToInteger` modulo 2^52, an die Windows-Runtime übergeben.  Wenn bei der Typkonvertierung ein Fehler auftritt, wird eine Ausnahme zurückgegeben.  |  
| `Single` | `Number` | Bei einer Windows-Runtime handelt es sich um eine 32-Bit-Gleitkommazahl, die als JavaScript-Zahl dargestellt wird, indem der nächste Wert mit doppelter Genauigkeit zum Wert mit einer `Single` einzelnen Genauigkeit verwendet wird.  Ein JavaScript-Wert wird in eine JavaScript-Zahl konvertiert und dann der Bereich überprüft, um sicherzustellen, dass der Wert in einer 32-Bit-IEEE 754-Gleitkommazahl mit einer Genauigkeit dargestellt werden kann.  Wenn die Konvertierungs- oder Bereichsüberprüfung fehlschlägt, wird ein Marshallingfehler zurückgegeben.  |  
| `Double` | `Number` | Eine Windows-Runtime `Double` ist eine 64-Bit-Gleitkommazahl.  `ToNumber` wird verwendet, um eine Windows-Runtime `Double` in eine JavaScript-Nummer zu konvertieren.  Wenn die Konvertierung fehlschlägt, wird ein Marshallingfehler zurückgegeben.  |  
| `Boolean` | `Boolean` | Eine Windows-Runtime ist ein 8-Bit-Wert, bei dem Null und jeder andere Wert als `Boolean` Null ist , dargestellt als `false` `true` JavaScript `Boolean` .  Ein JavaScript-Wert wird zuerst von in ein JavaScript `Boolean` `ToBoolean` konvertiert.  Dies bedeutet, dass eine Windows-Runtime-Funktion, die als deklariert `void func(BOOL);` wurde, in JavaScript als geschrieben werden `obj.func("test");` könnte.  Die Windows-Runtime-Funktion wird aufgerufen, wenn der `BOOL` Parameter als übergeben `TRUE` wird.  Wenn die Konvertierung fehlschlägt, wird ein Marshallingfehler zurückgegeben.  |  
| `Char16` | `String` | Eine Windows-Runtime `Char16` ist ein 16-Bit-Unicode-Zeichen, das als JavaScript-Zeichenfolge dargestellt wird, die ein einzelnes Zeichen enthält.  Ein JavaScript-Wert wird von in eine JavaScript-Zeichenfolge konvertiert und dann überprüft, um sicherzustellen, dass die Zeichenfolge nur ein einzelnes `ToString` Zeichen lang ist.  Das einzelne Zeichen wird dann als Wert `Char16` übergeben.  Wenn die Konvertierungs- oder Längenüberprüfung fehlschlägt, wird ein Marshallingfehler zurückgegeben.  |  
| `String` | `String` | Eine Windows-Runtime `String` ist eine unveränderliche Abfolge von `Char16` Objekten.  Zeichenfolgen werden als JavaScript-Objekte `String` dargestellt.  Windows-Runtime-Zeichenfolgen haben keine unterschiedlichen Werte für und `null` leere Zeichenfolge \(""\).  `null` und leere Zeichenfolgenwerte werden in die leere JavaScript-Zeichenfolge konvertiert.  **HINWEIS**: Wenn JavaScript an einen Windows-Runtime-Parameter übergeben wird, der eine Zeichenfolge erwartet, wird der Zeichenfolgenwert "null" oder die leere Zeichenfolge `null` `null` \(""\) erzeugt.  Zeichenfolgen, die an die Windows-Runtime übergeben werden, sollten immer an die leere Zeichenfolge instanziiert werden.  |  
| `Enumeration` | `Object` | Eine Windows-Runtime ist eine 32-Bit-Ganzzahl \(signiert oder nicht signiert\), der eine Reihe benannter `Enumeration` Konstanten zugeordnet ist.  In JavaScript wird eine Enumeration als Objekt dargestellt, das ein schreibgeschütztes Feld für jeden benannten Wert enthält.  Enumerationswerte werden in JavaScript-Zahlen konvertiert, wie zuvor für und `Int32` `UInt32` definiert.  Wenn ein JavaScript-Wert in eine Windows-Runtime-Enumeration konvertiert wird, wird er wie zuvor beschrieben konvertiert, abgeschnitten und der Bereich überprüft.  Der resultierende Wert wird jedoch nicht mit den definierten benannten Werten überprüft, die in der Enumeration angegeben sind.  |  
| `Structure` | `Object` | Eine Windows-Runtime `Structure` ist eine heterogene Auflistung benannter Datenfelder.  In JavaScript wird eine Struktur als JavaScript-Objekt dargestellt, das eine benannte Eigenschaft für jedes Feld in der Struktur enthält.  Wenn bei der Konvertierung eines Strukturwerts ein Fehler auftritt, wird ein Marshallingfehler zurückgegeben.  Felder im JavaScript-Objekt, die keine Entsprechungen in der Windows-Runtime-Struktur haben, werden ignoriert.  **HINWEIS:** Eine Windows-Runtime-Struktur kann nicht in JavaScript mit dem Schlüsselwort `new` instanziiert werden.  |  
| `Array` | `Array` | Eine Windows-Runtime `Array` wird in ein JavaScript-Array konvertiert.  Windows-Runtime-Arrays können jedoch nicht in der Größe gespeichert werden, sodass einige JavaScript-Arrayvorgänge nicht möglich sind.  Die interne Eigenschaft eines konvertierten Arrays ist nicht auf "Array" festgelegt, da dies in der `[[Class]]` ECMAScript 5-Spezifikation nicht zulässig ist.  Dies bedeutet, dass `Array.isArray(v)` zurückgegeben `false` wird.  Ein JavaScript-Wert wird wie folgt in ein Windows-Runtime-Array konvertiert: Wenn der Wert oder ist, wird `null` er in ein `undefined` systemeigenes `null` konvertiert.  Wenn die interne Eigenschaft "Array" ist, wird sie in ein systemeigenes Array kopiert, und ein `[[Class]]` Verweis auf dieses Array wird übergeben.  Wenn es sich wie zuvor beschrieben um ein konvertiertes Array handelt, wird das zugrunde liegende systemeigene Array übergeben.  **HINWEIS**: Das Übergeben eines JavaScript-Arraywerts an eine Windows-Runtime-Methode führt zu einer vollständigen Kopie des Arrays.  |  
| Delegate \(function callback\) | `Function` | Ein Windows-Runtime-Delegater ist ein Verweis auf eine einzelne Methode.  Ein Windows-Runtime-Delegate wird in JavaScript als JavaScript dargestellt, das garantiert im `Function` richtigen Thread aufgerufen wird.  Wenn der `Function` aufgerufen wird, werden die Argumente in die entsprechenden Windows-Runtime-Parametertypen konvertiert.  Wenn weniger JavaScript-Argumente als Stellvertretungsparameter angegeben `in` sind, schlägt der Delegataufruf fehl.  Wenn mehr JavaScript-Argumente als Parameter im Delegaten enthalten sind, werden die `in` zusätzlichen JavaScript-Argumente ignoriert.  Nachdem der Delegat aufgerufen wurde, werden `out` die Parameter in JavaScript-Typen konvertiert und zurückgegeben.  Wenn ein systemeigenes JavaScript-Funktionsobjekt in einen Windows-Runtime-Delegaten konvertiert wird, wird es in einen Windows-Runtime-Delegaten des entsprechenden Typs umschlossen.  Wenn der Delegat aufgerufen wird, werden `in` die Parameter in JavaScript-Typen konvertiert.  Nachdem der Delegat aufgerufen wurde, wird der Rückgabewert in die `out` Parameter des Delegaten konvertiert.  |  
| Schnittstelle | `Object` | Schnittstellen und Schnittstellengruppen werden in JavaScript nicht direkt als Objekte dargestellt.  Schnittstellen werden jedoch als Parameter- und Rückgabetypen von Windows-Runtime-Methoden dargestellt.  Eine Windows-Runtime-Schnittstelleninstanz wird wie folgt konvertiert:<br /> <br /> 1. Wenn eine gültige Laufzeitklasse vor ist, stellen Sie das Objekt als Instanz dieser Klasse dar.<br /> 2. Wenn keine gültige Laufzeitklasse verfügbar ist, stellen Sie das Objekt als Instanz einer unbenannten Laufzeitklasse dar, die genau die Schnittstelle implementiert, die die Instanz sowie die erforderlichen Schnittstellen implementieren kann.<br /> <br /> Ein JavaScript-Wert wird getestet, um zu ermitteln, ob es sich um eine Instanz einer Laufzeitklasse oder einer Schnittstelle handelt.  Wenn dies der Zustand ist und der ursprüngliche Windows-Runtime-Wert die Schnittstelle implementiert, wird das Objekt an die Windows-Runtime übergeben.  |  
| Laufzeitklassen | `Object` | Objekte in der Windows-Runtime können Instanzen von Laufzeitklassen sein, die eine oder mehrere Schnittstellen implementieren.  Windows-Runtime-Objekte werden in JavaScript als Objekte dargestellt.  Die Vereinigung von Methoden, Eigenschaften und Ereignissen, die für alle implementierten Schnittstellen der Klasse definiert sind, stellt die benannten Eigenschaften im Prototyp des JavaScript-Objekts dar.  JavaScript-Benutzer können direkt auf jedes Element der Klasse zugreifen.  Windows-Runtime-Objekte verfügen über einen Prototyp, der alle Member aller implementierten Schnittstellen der Laufzeitklasse enthält.  |
